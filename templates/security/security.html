SECURING THE DIGITAL MANUFACTURING COMMONS

We are committed to our security practices and helping you understand our approach.

SECURITY: HOLISTIC APPROACH

DMC is leading the industry in our security approach to protect digital manufacturing. We have adopted NIST 800-53 and ITAR frameworks improving security controls above the industry standards.

People

Our team of professionals meets rigorous internal requirements, policies, and standards.

Security and Privacy Training

The entire organization undergoes annual security and privacy training, and DMC team members go through monthly security training sessions covering privacy, security topics (including device security), acceptable use, preventing malware, physical security, data privacy, account management, and incident reporting. Employees are informed that failure to comply will result in consequences, up to and including termination.

SECURITY: DEDICATED PRACTICES

DMC has defined roles and job responsibilities dealing with all aspects of securing the environment, platform and organization.

The security program covers:

  • Establishing secure development practices and standards
  • Training developers on secure coding practices
  • Code auditing – detection, removal, and mitigation of coding security flaws
  • Event monitoring
  • Security logging
  • Secure deployment and configuration management
  • Managing, responding to, and resolving security Incidents
  • Penetration / vulnerability testing and remediation
  • ITAR compliance (for detailed information please see ITAR compliance section)

SECURITY: COMPLIANCE THROUGH AUDITING

DMC’s security program is designed to address security standards as outlined by NIST 800-53 and ITAR frameworks. We meet these thresholds through both internal and third party auditing and penetration testing.

Auditing

Third party audits are performed annually. Internal audits are performed following the continual release platform cycle (quarterly, at minimum).

Penetration Testing

Infrastructure and web application testing is performed by the DMC team monthly, with third party auditing performed as needed (annually, at minimum).

SECURITY: SOFTWARE DEVELOPMENT LIFECYCLE

Our team of developers are required to check all code into a version-controlled repository, with all code changes undergoing peer review and an approval process before implementation. Static analysis is performed using industry-leading tools and techniques.

SECURITY: PROTECTING MEMBERS' DATA

DMC’s security program is focused on protecting our members’ IP and data by implementing best practices, identifying and mitigating risks, and constantly evaluating ways to improve our security.

Data Encryption

Data is encrypted in transit and at rest, utilizing the latest recommended security cyphers, encryption, and transport technologies (TLS, AES256, SHA2 signatures, FIPS 140-2 encryption standards).

AZURE

With the platform hosted in AZURE, Microsoft provides the following additional security metrics: secure datacenters, audit trails, video 24/7, and software defined perimeters.

Cloudflare OWasp Protection

Cloudflare provides identification of vulnerabilities such as cross site scripting, SQL injection, comment spam, excessive bot crawling, and DDoS attacks.

Authentication

Platform authentication is achieved through federated services that utilize OAuth, SAML, and ADFS.

Monitoring

System monitoring, logging, and alerting is achieved using a multipoint approach. The DMC has integrated security at all levels capturing all relevant data such as ELK, Suricata, and Cloudflare.

Server Hardening

Standards have been implemented with our automated deployments preventing accidental misconfiguration resulting in vulnerabilities and improving consistency, reliability, and manageability of all systems deployed to the platform.

SECURITY: ITAR COMPLIANCE

The DMC platform provides access controls and accountability features that enable its users to enforce ITAR requirements for control of transfer to dual nationals or third-country national (22 CFR Part 126.18). DMC’s support for 22 CFR Part 126.18 includes the following supporting functions:

22 CFR Part 126.18 (a)

DMC restricts use of the DMC environment and its components to individuals who have unique user accounts and passwords. This feature is employed using federated authentication and allowing companies to directly manage their assets.

22 CFR Part 126.18 (b)

DMC prevents the export of components of the platform, including scripts, object definition files, configuration files, and APIs, to assigned end-users. These configurable permissions are employed using digital rights management at upload time.

22 CFR Part 126.18 (c) (1)

This requirement is governed by the DMC Member, and requires the Member to process security clearances of their assigned users, as appropriate. Upon providing the assigned user the appropriate clearance, appropriate permissions will be apportioned using the features described in 22 CFR Part 126.18(a) and 22 CFR Part 126.18(b).
